## V2-2 — HIGH (latent) — Funnel sync re-arms sequences with STALE clocks
funnelSync.js:114-115: on status promote, sync copies seqTrack/seqStep/seqPaused from mapFunnelStatus — including seqPaused:false — WITHOUT resetting seqStartDate. Re-armed leads with old start dates compute every touch as past-due → next 9AM cron fires a backlog of EMAILS at them (SMS is kill-switched; email is not). This is the same failure shape as the 6/11 incident, email edition. The 6/11 4AM burst was partly THIS: the 6/10 sync re-armed ~300 leads.
Fix (1 line + doctrine choice): when sync unpauses a sequence, set seqStartDate = now (drip restarts from step 0 cleanly), OR never let sync touch seqPaused (status ≠ enrollment). NEEDS JEREMY: should a Funnel status change enroll a family in the Metka email drip at all?

## V2-3 — MED — receive-sms security holes
a) Signature validation FAILS OPEN (`catch { return true; }`) and is SKIPPED ENTIRELY if TWILIO_AUTH_TOKEN secret is unset. The URL is public + verify_jwt false → anyone who finds it can forge inbound SMS: mass-flag smsOptOut (kills your marketing list), inject fake threads, flip opt-INs. Fix: fail closed; require the secret; log+reject invalid signatures.
b) send-sms has NO server-side optOut/quiet-hours check — it trusts every caller. Add DB lookup: reject if lead.smsOptOut (defense in depth for all current/future callers).
c) Performance: receive-sms table-scans ALL 2,447 leads per inbound message (paged 1000s). Works; wasteful. Fix later: promoted phone column + index.

## V2-4 — MED — Live RLS confirmed wide open + anon key exposure
pg_policies LIVE check: anon role has SELECT/INSERT/UPDATE/DELETE with qual=true on leads AND activity. Anyone with project URL + anon key can read or WIPE the database today. The anon key is embedded in plaintext in the cron.job command (visible to any SQL access) and ships in every app bundle. Currently private/localhost = contained; remains the hard deploy gate. If the repo or bundle ever leaks: rotate keys immediately.
